Git

  
一、创建git仓库流程

1、新建一个项目文件进去右键执行Git Bash

2、输入git init

3、在config中添加下面代码，否则不能push
       [receive]
       denyCurrentBranch = ignore

4、随便添加相关文件

5、执行git add */-A/file.txt   (结果将会保存到本机的缓存里面)

6、git commit -m'注释'         (提交，把本机缓存中的内容提交到本机的 HEAD 里面)

7、仓库创建完成

8、检出创建的仓库
  （1）git clone/pash克隆仓库
       本地：cd e:             (跳到e盘下)
                git clone /g:aa   (/后面为被克隆的仓库的位置)
                cd 克隆完的仓库名称
       远程：cd e:             (调到e盘下)
                git clone 远程的地址（git@github.com:19880229/project.git 自己想用的名字）
	              如果不执行这个命令的话，每次 push 的时候都需要指定远程服务器的地址
	 cd 克隆的远程库名称
  （2）添加文档
  （3）执行“git add file.txt”   
  （4）执行git commit -m"说明"  
  （5）git push 



二、常用命令

1、配置git
   git config --lish  检查已有的配置信息，没有则执行下面命令
   git config --global user.name'xian509'
   git config --global user.email'1041623374@qq.com'

2、配置 sshkey ：
   (1)输入：ssh-keygen -t rsa -C "1041623374@qq.com" （连续回车三次）
   (2)在 Github 中添加这个 sshkey：
      登录 Github --> Account Setting  --> SSH-KEY --> Add SSH-KEY --> 粘贴id_rsa.pub中的内容；
   (3)验证:
      ssh -T git@github.com
      出现 Hi xxx! You've successfully authenticated, but GitHub does not provide shell access. 说明配置成功，可以连接上 Github

3、初始化、提交、拉取相关命令
   git init                 (初始化仓库，若加“--bare”，则初始化裸仓库)
   git clone 仓库地址(克隆仓库)  本地仓库名称
   git add -A               (保存所有改动)
   git commit -m'注释'      (提交到本地)
   git push [origin] 分支名称        (提交到远程分支)
   git pull [origin] 远程名称:本地分支名称  （如果省略本地分支名，则表示远程分支是与当前分支合并）
   git fetch 分支名称       (拉取远程库中所有没有的数据及分支)

4、分支
  （1）、查看分支：
                 git branch     (只查看本地分支)
		 git branch -r (只查看远程分支)
		 git branch -a  (查看本地和远程分支)
  （2）、创建分支：
                 git branch 分支名称      （创建本地分支）
		 git checkout 分支名称    （切换到此分支）
		 git checkout -b 分支名称 （创建并切换此本地分支）
  （3）、打开分支：
                 git checkout 分支名称    （本地分支）
  （4）、提交到远程：
                 git push origin 分支名称  (本地分支提交到远程)
                 git push --set-upstream origin 分支名称   （提交本地分支到远程）
  （5）、删除分支：
                 git branch -D 分支名称             （删除本地分支）
		 git push origin --delete 分支名称  （删除远程分支）

5、查看每个人分支最后一次提交
       git branch -v
